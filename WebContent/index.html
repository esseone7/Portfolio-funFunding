5. 화면 구현 내용 - 메이커 스튜디오


5-1. 메이커 등록




5-1-1. 필수 동의 사항 체크해야 메이커 등록(시작하기) 가능
5-1-2. MAKER 기본 정보 DB에 INSERT (차후 단계에서 MAKER 상세 정보 수정 가능)


5-1-3. 프로젝트 등록 신청 날짜 기준으로 프로젝트 등록을 수정 혹은 프로젝트 관리 페이지로 이동




5-2. 프로젝트 등록




5-2-01. 각 항목에 입력된 값을 체크하여 입력값이 존재하면 ‘작성완료’, 입력값이 NULL 또는 0이면 ‘작성전’ 표시
5-2-02. 모든 항목이 작성 완료가 되면 하단에 프로젝트 등록 신청 버튼 표시
5-2-03. 등록 신청 시 DB에 등록 날짜를 UPDATE하여 더 이상 수정 불가 및 승인 될 때까지 대기


5-2-04. MultipartResolver를 이용하여 이미지 파일 업로드를 위한 Uploader util 생성 후 사용
5-2-05. 서버에 등록된 파일의 이름이 중복되는 것을 방지하기 위해 UUID.randomUUID()를 사용하여 임의의 파일명을 생성하여 서버에 업로드


5-2-06. 웹에디터 CKEDITOR를 활용하여 프로젝트의 상세 내용 소개를 등록하는데 자율성 부여
5-2-07. 이미지 업로드 가능, 작성된 내용은 html 태그 형식으로 DB에 업로드


5-2-08. 다수의 리워드를 등록 가능하며, 펀딩 종료 및 목표 펀딩 금액 달성 후 오픈하는 스토어의 옵션에 1:1로 매치 됨 5-2-09. X 버튼을 이용하여 삭제 및 리워드 수정 버튼을 이용하여 수정 가능 5-2-10. 수정 및 삭제 내용은 DB에 반영



5-3. 프로젝트 관리




5-3-1. 프로젝트 등록 신청 날짜, 프로젝트 승인 날짜, 프로젝트 시작 날짜, 프로젝트 종료 날짜, 달성률을 기준으로 현재 상태 표시


5-3-2. 등록된 답변을 기준으로 화면 UI 변경
5-3-3. 답변이 등록되지 않은 경우 답변 등록 버튼 노출, 답변이 등록된 경우 등록된 답변과 함께 답변 수정 버튼 호출




5-4. 스토어 등록




5-4-1. 각 항목에 입력된 값을 체크하여 입력값이 존재하면 ‘작성완료’, 입력값이 NULL 또는 0이면 ‘작성전’ 표시
5-4-2. 모든 항목이 작성 완료가 되면 하단에 스토어 등록 신청 버튼 표시
5-4-3. 등록 신청 시 DB에 등록 날짜를 UPDATE하여 더 이상 수정 불가 및 승인 될 때까지 대기


5-4-4. LEFT JOIN을 활용하여 프로젝트의 리워드와 스토어의 옵션을 1:1로 매칭하여 등록할 수 있도록 설계




5-5. 리워드 판매 관리




5-5-1. 스토어 1개에 다수의 옵션을 설정할 수 있고, 각 옵션에 주문을 생성하므로, WHERE, IN을 활용하여 주문 관리 LIST 출력
